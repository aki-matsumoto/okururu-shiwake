# オクルウル 仕分け支援ツール (ReCORE) v0.3.0

買取ケース詳細ページ（`https://app.recore-pos.com/bad/case/<id>`）で、仕分けに必要なマークを自動表示し、ITコードラベルの個別/一括印刷を補助する Chrome 拡張。**既存「査定支援拡張」とは別物の独立拡張**です。

## マーク一覧

| バッジ | 条件 |
|--------|------|
| 📦 卸 | 商品名の先頭が「□」 |
| 🏯 リコア | 商品名の先頭が「■」 |
| 🏬 デパコス | デパコス39ブランド（H列〇）に一致 かつ 化粧品/香水系カテゴリ |
| 🚫 メルカリ禁止 | メルカリ出品禁止13ブランドに一致（カテゴリ問わず） |
| 📌 ITコード | 買取¥3,000以上(非コスメ系)／貴金属¥3,000以上／販売¥4,000以上(全カテゴリ)／スマホ・タブレット¥1以上／ブランド品 |

- デパコス かつ メルカリ禁止（シャネル/ディオール/クレ・ド・ポー/エルメスB/プラダB/グッチB）は **両バッジ** が付きます。
- 📌 ITコード行には「🖨 個別印刷」ボタン、ページ下部に「🖨 ITコード一括印刷（N枚）」ボタンを表示。

## v0.2.0 追加（2026-06-15 実機検証反映）

- **マーク・ボタンは商品名の「上」にまとめて表示**（マークバー）。卸/リコア・デパコス・メルカリ禁止・ITコード・個別印刷ボタンが1行に並ぶ。
- **不要項目を非表示**：棚番号／他店価格／在庫属性（popupでON/OFF、既定ON）。
- **重複警告**：既存「査定支援拡張」が同時稼働していると上部に赤い警告バーを表示（DOM footprintで検知）。拡張は他拡張を自動OFFにできないため、chrome://extensions で手動OFFを促す。実機確認：査定支援を切っていれば警告は出ない。

## v0.3.0 追加：アップデート通知

- `background.js` が **30分ごと**に `release/latest.json` を取得し、現バージョンより新しければ上部に青い通知バー（「今すぐ更新」ボタン付き）を表示（s7）。× で当該版は非表示。
- `latest.json` に `"force_min_version"` を入れると、それ未満のPCは自動で `chrome.runtime.reload()`（全PC強制更新）。
- **配布前にやること**：フィードURL `https://raw.githubusercontent.com/aki-matsumoto/okururu-shiwake/main/release/latest.json` の GitHubリポジトリ（`okururu-shiwake`）を作成し、`release/latest.json` を main に置く。リポ名を変える場合は `background.js` の `VERSION_FEED_URL` と manifest の `host_permissions` を合わせて変更。
- 既存「査定支援拡張」と同様 push-feature.ps1 ＋ CI 自動タグ運用に載せれば、リリース＝latest.json更新で全PCに通知が飛ぶ。

## インストール（load unpacked）

1. Chrome で `chrome://extensions` を開く
2. 右上「デベロッパーモード」をON
3. 「パッケージ化されていない拡張機能を読み込む」→ この `okururu-shiwake` フォルダを選択
4. ReCORE の買取ケース詳細ページを開く（既に開いていれば再読み込み）

## ポップアップ（機能ON/OFF）

ツールバーのアイコンから 4 機能を個別にON/OFF。変更後はReCOREタブを再読み込みすると確実に反映。

## 印刷の仕様（2026-06-15 実機検証で確定）

- ITコード＝**在庫ラベル**。印刷は「各行の『ラベル印刷』ドロップダウン項目」を click する（ページ全体ボタンのN回clickではない）。既存 F79 と同方式。
- **ITコードは買取確定(STOCKED)後にのみ生成・印刷可能**。未確定の間は一括/個別ボタンは無効（tooltipで案内）。
- 個別＝その行の「ラベル印刷」を1回 / 一括＝📌ITコードマーク行の「ラベル印刷」を500ms間隔で順次。確認モーダル＋ESC中止。
- 注意：既存「査定支援拡張」にも F78(ケースラベル一括)/F79(ITコード一括) があるため、両方インストールするとボタンが重複表示されます。

## データ更新

`data/shiwake_brands.js` の `DEPACOS39` / `MERCARI_BAN13` を編集すれば対象ブランドを増減できます（patterns に商品名で出やすい英字/カナ表記を追加）。
